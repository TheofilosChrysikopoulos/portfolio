About this project

I built this project for fun—and out of frustration. Early on, I struggled to create attractive environments, and most terrain tools felt menu-heavy and unintuitive. My idea was a VR 3D World Creator that lets anyone build a world quickly with straightforward, tactile interactions, then explore it in first person. The goal was an experience simple enough that even a newcomer could produce something beautiful.

This was my first VR project. I designed world-positioned UI panels that live in the user’s space and can be placed wherever they’re convenient, which also pushed me to get comfortable with Unity’s UI system. For terrain, I implemented a marching-cubes pipeline powered by compute shaders, and wrote shaders to handle texture painting directly on the surface. I explored both procedural and manual approaches to object placement and shipped a lightweight tool for dropping models into the scene. To support physics, I added a method that converts an arbitrary 3D model into a compressed set of convex shapes—ideal for efficient mesh colliders and aligned with Unity’s preference for convex hulls in physics and collision.

What I learned

I learned how to design VR-first interactions, build spatial UI, and balance visual fidelity with performance on VR hardware. Implementing marching cubes with compute shaders, for the terrain mesh, sharpened my understanding of GPU-driven workflows, while writing terrain/painting shaders improved my grasp of material systems. Extending the editor with custom placement tools—and implementing convex decomposition for colliders—taught me a lot about practical level-building pipelines and physics constraints in Unity.

The project is unfinished, but it stands as a proof of concept: creating a rich environment quickly and comfortably in VR is absolutely achievable. Future goals include save/load for scenes, undo/redo, a larger model library, user model import, and additional terrain editing tools.
